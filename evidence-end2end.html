海关案件证据全链路规划 - 方式一：数据格式化

主线： 案件建立/权限 → 证据管理 → 数据格式化 → 证据挖掘 → 证据复核
子阶段映射： 上传/目录 → 清洗 → 标签/检索/比对 → 输出/审计
目标： 完整采集、多模态可计算、智能挖掘、可追溯复核与合规交付。
系统底座： 组织/人员、元数据/字典、审计/监控，统一支撑全流程的权限、命名空间与合规治理。

待验证- 方式二： 采用skill or Agent 直接用大模型进行筛选和挖掘。不再用格式化，利用大模型的多模态能力。

证据挖掘处理逻辑（证据大表填充逻辑）

整体定位与主线嵌入

证据大表是“还原犯罪过程证据链路”的核心产物，应当在案件生命周期中被明确为全流程的填充目标。采用“两段式”方案：在 0.案件建立/权限 阶段完成“案型—证据大表模板”的绑定；在 4.证据挖掘 阶段生成草表；在 5.证据复核/输出 阶段定稿。0 阶段由案型驱动模板字段、口径、时间线维度、关键参与者维度与资金/物流/通讯链路结构，并纳入案件元数据与审计。3.数据格式化阶段产出“可映射字段”的中间结构；4.证据挖掘阶段完成字段填充、冲突消解与置信度计算，生成草表；5.阶段执行人工复核与签名定稿，保证司法可用与可追溯。

字段映射与填充机制

建议引入“字段映射层（Field Mapping Layer）”，为每个模板字段定义口径、来源优先级、允许证据类型与冲突规则。例如“交易时间”允许来自支付流水/聊天记录/物流签收等来源，并对精度（分钟/天）打标。数据格式化阶段输出“字段候选池”（含来源证据ID、提取方式、置信度、时间戳、主体/客体、原文片段与结构化值）。证据挖掘阶段完成候选融合与异常标注，并记录“填充链路（Fill Lineage）”：字段 → 候选 → 证据 → 原始文件；同时维护字段状态（空缺/已填/需复核/冲突），让复核人员聚焦风险字段。

版本链、审计与权限联动

证据大表需与证据管理的“不可变 + 版本链”机制联动。每次字段更新（自动或人工）生成新版本节点，记录变更原因、字段清单、审批人与差异摘要，并与证据版本 hash/manifest 建立双向引用。权限上，大表字段具备粒度化控制：调查员可填充字段但不可覆盖高风险字段，复核人员拥有冲突字段最终决策权，管理员有回滚权。证据版本发生迭代时触发“影响评估”，标记受影响字段为“需复核”，生成差异报告；回滚策略为“回滚即新版本”，保证历史可追溯与审计闭环。

案型模板差异与证据大表填充

针对不同案型，证据大表应采用“模板驱动、分表组织、字段映射一致”的结构。以“境外走私代购（奢侈品代购类）”为例，证据大表拆为“销售情况/购买情况”两类输出表，字段聚焦交易链路：交易方、商品名称/型号/数量、报价与付款金额分解（微信/支付宝/银行卡/其他）、交易时间、发货与物流（发货信息、运输公司、运单号）、保卡号、过货人/时间、收货信息与备注等，形成“交易 → 物流 → 收货”的证据链路可复原表。对“套代购”则保留两套逻辑：其一“以时间和任务关系为准”的多源匹配型大表，强调免税店购物数据 ↔ 支付数据 ↔ 微信/支付宝/银行转账数据的时间差、付款差、跨小票关联、匹配等级与积分扣减；其二“以购买商品为准”的商品粒度分析表，强调商品数量不一致、应付/实付差异、两次删除未计税数据对比、人工/AI 匹配差异原因等。两套表同时输出“已匹配/未匹配”“支付全量列表/购物人员列表”，支撑复核与补证。

整体流程图（含分支 + 系统底座）

按处理流程描述

0.系统底座能力

  • 组织与人员： 关区/科室/小组组织树，岗位/角色模型，人员主数据（职务、警号、联系方式），支持多级授权与代理。
  • 元数据 / 字典： 案件类型、关区、HS 编码、渠道、风险标签、证据类别、保密级别等统一字典；版本化与发布流程；API/前端表单统一引用。
  • 统一权限与命名空间： 租户/案件双层隔离，命名空间规范（caseId 前缀），跨案访问需授权与审计。
  • 审计 / 监控 / 合规： 统一日志与审计流水；安全基线（MFA、IP 白名单、操作水印）；指标与告警（权限异常、未授权拦截、性能指标）；合规策略（留存、冷存、合规锁）。

1.案件建立 / 权限（主线起点）

这一段确保“正确的人在正确的案件里做正确的事”，为后续证据链路奠定安全与治理基线。

  • 案件生命周期： 创建 → 分配责任人/团队 → 设置案件元信息（类型、关区、涉案级别、保密等级、留存策略）→ 状态流转（草稿/进行中/结案/归档/封存）。
  • 案件类型： 奢侈品走私、套代购、虚开、走私冻品等；类型驱动目录/命名模板、必填元数据（申报单号、物流单号、航次/航班、商品 HS 编码等）、默认保密级别、清洗/挖掘规则集与配额策略。
  • 证据大表模板绑定： 在立案阶段完成案型-模板绑定，明确大表字段口径、字段映射规则、必填字段与版本策略，为后续填充与复核提供统一目标。
  • 案情基础人物关系图谱： 案情获取时形成初版关系图谱（人-人/人-物/人-地点），作为 3/4 阶段共享资产持续完善，图谱版本与来源证据纳入审计。
  • 身份与授权： 角色（管理员/案件负责人/调查员/审计/只读）+ 权限点（上传、目录、标签、搜索、导出、复核、规则管理）；支持临时授权、时间窗与范围限制。
  • 隔离与命名空间： 案件即租户边界；存储/索引/向量库/日志均按 {caseId} 分区；跨案访问需显式授权并审计。
  • 安全基线： 登录与 MFA，IP/设备指纹，会话有效期，操作水印；高风险操作（删除/导出/规则发布）可启用双人校验或二次审批。
  • 审计与可观测： 案件级审计流水（用户、角色、动作、对象、结果、耗时、IP/UA）；指标：活跃用户数、权限变更次数、未授权拦截；Trace 从网关入手贯穿案件上下文。
  • 合规与留存： 案件级保留/归档策略、冷存/合规锁；封存与解封流程可追溯；支持外部审计抽查。

2.证据管理

  • 采集：目录/批量/分片/预签名上传，REL_PATH 固化，batchId、caseId 绑定，MD5/ETag 去重，病毒扫描，配额/速率限制。
  • 目录与存储：案件隔离的虚拟文件系统（创建/移动/删除/回收站）、路径净化、幂等移动；本地/MinIO/S3 适配；容量统计；按案件类型加载目录/命名模板并校验必填元数据。
  • 不可变性与版本：原始证据默认不可更改；案件负责人可发起“版本迭代”申请（生成新版本并保留原版本哈希与时间戳），删除仅管理员可执行；所有版本链路与操作记录审计留痕。
  • 审计与可观测：上传/目录操作日志，Trace 覆盖网关→上传→存储，指标（成功率、P95、去重命中）。

3.数据格式化

目标是把多模态证据转为“可计算、可检索、可训练”的结构化数据。重设计为“能力颗粒库 + 编排层 + 策略配置”三层结构，3.1 高质量与 3.2 高效率是同一能力库的不同编排与开关组合。

能力颗粒库的详细清单、实现方式与分类落库映射统一放在 3.3.2 章节中，便于与落库策略融合管理。

3.1 template 简单案件

  • 核心选择： 文本抽取 + OCR/表格识别 + 实体/关系抽取 + 质量校验 + 落库归档。
  • IM 多模态： 文本/图片/语音/视频分流解析，OCR + ASR + 关键帧，重建对话时间线与事件序列。
  • 直接文件： PDF/图片全量 OCR + 表格识别；Excel 全量解析；EML 头/正文/附件解析。
  • 产出： 高完整度结构化候选池，可直接进入草表生成与复核。

3.2 template 复杂案件

  • 核心选择： 元数据抽取 + 轻量索引 + 时间线/参与者 + 初版关系图谱。
  • IM 轻索引： 先抽取参与者、时间线与消息摘要，多媒体仅建索引与元数据。
  • 定向深解析： 基于案情条件（时间/地点/人物/关系）触发 OCR/ASR/表格识别与实体抽取。
  • 多轮迭代： 条件变化 → 定向解析 → 草表更新，保留可追溯链路。

3.3 策略切换与数据落地（3级功能）

  • 按案件数据量、文件类型占比、风险等级或命中规则决定 A/C；支持“C → A”按需升级。
  • 字段候选池：输出字段候选（值/来源证据ID/提取方式/置信度/时间戳），为证据挖掘阶段的字段融合与草表生成提供输入。
  • 落地与治理：以统一命名空间落地到数据库层（SQL / 湖仓 / 向量库 / 图数据库），按 caseId/batch/ingest_date 分区；元数据索引与 schema 规划。
  • 监控：P95/P99、OCR 置信度、抽帧命中率，资源限流保障多租户稳定。

3.4 编排示例（简单/复杂）

  • 简单案件（一次性高质量）： 元数据抽取 → 文本抽取/OCR/表格识别 → 实体/关系抽取 → 质量校验 → 落库 → 字段候选池。
  • 复杂案件（分层+多轮）： 元数据抽取 → 轻量索引 → 时间线/参与者 → 初版图谱 → 条件筛选 → 定向深解析 → 迭代更新。
  • 关系图谱增量： 实体/关系抽取结果持续回写图谱，记录证据来源与置信度，形成可追溯的图谱版本。
  • 切换机制： 依据文件量、结构化占比与图谱复杂度动态调整 A/C 策略，触发“C → A”按需升级。

4.证据挖掘（增强 LLM / BI数据分析）

证据挖掘采用“按结构化程度分支”的闭环流程，结合规则、SQL/BI 与 LLM/图谱推断，服务证据大表字段填充与复核闭环。

挖掘分支处理闭环（按数据结构化程度）
  • 高结构化（表格/票据为主）： SQL/BI 主导 → 规则匹配/聚合/对账 → 异常识别 → 证据大表字段填充 → 规则阈值回写与口径修订。
  • 混合型（表格 + IM/多模态）： SQL/BI 与 LLM 并行 → 结构化字段对齐 + 语义抽取融合 → 关系图谱校验/补全 → 大表填充与置信度评分 → 回写规则与提示模板。
  • 低结构化（IM/多模态为主）： LLM 语义理解 + 图谱推断 → 事件链路/人物关系生成 → 定向触发深解析（OCR/ASR/关键帧）→ 大表填充 → 多轮条件调整与图谱增量更新。

分支选择依据：结构化占比、字段缺失率、图谱复杂度与案情要求；支持“低结构化 → 混合型 → 高结构化”按需升级。
